# Snake Arcade GitHub Pages Site

This repository is meant to be created as:

`platano91.github.io`

That makes GitHub Pages publish the site at the account root hostname instead of under a project
subpath.

## Files To Keep At Repository Root

- `index.html`
- `privacy-policy.html`
- `app-ads.txt`
- `styles.css`
- `.nojekyll`

## Enable GitHub Pages

1. Create or open the public repository named exactly `platano91.github.io`
2. Push these files to the `main` branch
3. On GitHub, open the repository
4. Go to `Settings`
5. In the sidebar, click `Pages`
6. Under `Build and deployment`, set `Source` to `Deploy from a branch`
7. Set `Branch` to `main`
8. Set `Folder` to `/(root)`
9. Save

## Expected Live URLs

- `https://platano91.github.io/`
- `https://platano91.github.io/privacy-policy.html`
- `https://platano91.github.io/app-ads.txt`

## Important Reminder

Use the same hostname in your App Store / Play Store developer website fields:

`https://platano91.github.io`

That keeps the privacy-policy URL and the AdMob `app-ads.txt` hostname aligned.
